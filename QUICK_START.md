# 🚀 퀵스타트 가이드

## 🎯 5분 만에 시작하는 AI주도개발 시스템

이 가이드는 구매 후 즉시 시스템 프롬프트를 활용할 수 있도록, 핵심적인 사용법을 5–10분 안에 설명합니다.

## 📋 사전 준비 (2분)

### 필요한 도구
- **Cursor IDE**: 최신 버전 설치
- **프로젝트 폴더**: 개발 대상 프로젝트 준비
- **기본 개발 환경**: Node.js, Python 등(프로젝트에 맞게)

### 파일 배치
1. 다운로드한 폴더 압축 해제
2. `.cursor/rules` 또는 `.cursor/commands` 폴더에 배치
3. Cursor에서 프로젝트 열기

## 🎯 처음 3단계 (3분)

### Step 1: 개인 개발인지 기업/팀 개발인지 결정

**개인 개발/취미 프로젝트의 경우**
- Phase 3.3(대화형 요구정의)부터 시작
- `03_solution_design_evaluation/03-interactive-requirements-definition.md` 사용

**기업/팀 프로젝트의 경우**  
- Phase 1(과제 이해)부터 전체 플로우 실행
- `01_challenge_understanding/01-initial-interview.md` 사용

### Step 2: 첫 프롬프트 실행

**개인 개발 예시**
```
@03-interactive-requirements-definition.md를 사용해, 아래 프로젝트의 요구사항을 정의해주세요:

[당신의 프로젝트 개요 입력]
예: "할 일 관리 앱을 만들고 싶다. 지금은 손글씨 메모로 관리하지만, 디지털화로 효율을 높이고 싶다."
```

**기업 개발 예시**
```
@01-initial-interview.md를 사용해, 아래 프로젝트의 초기 인터뷰 설계를 진행해주세요:

[프로젝트 배경 입력]
예: "영업 부서의 거래 관리 효율화를 원한다. 현재 Excel로 관리 중인데, 정보 공유와 진행 파악에 문제가 있다."
```

### Step 3: 대화 진행

- AI가 질문하면 최대한 구체적으로 답변
- 모르는 부분은 솔직히 "모름"으로 전달
- 점진적으로 이해를 깊게 만들어 가기

## 🔥 즉시 효과를 내는 핵심 프롬프트 4개

### 1. 대화형 요구정의 (Phase 3.3)
**사용시점**: 프로젝트 시작 시, 요구가 모호할 때
**효과**: 생산성 5배 핵심
**파일**: `03-interactive-requirements-definition.md`

### 2. Make/Buy판단 (Phase 2.3)
**사용 시점**: 기술 선정, 외부 서비스 검토 시
**효과**: 의사결정 객관화, 비용 최적화
**파일**: `03-make-or-buy-analysis.md`

### 3. 아키텍처 결정 (Phase 3.4)
**사용 시점**: 기술 스택 및 설계 방침 확정 시
**효과**: 선택 근거 명확화
**파일**: `04-architecture-decision.md`

### 4. 인지과학적 디버깅 (Phase 7.1)
**사용 시점**: 에러 발생, 학습 기반 디버깅
**효과**: 문제 해결 능력 향상, 재발 방지
**파일**: `01-debug-cognitive.md`

## ⚡ 효율적인 사용 팁

### 시간 절약
1. **프롬프트 부분 사용**: 필요한 부분만 발췌 사용
2. **템플릿화**: 자주 쓰는 패턴은 템플릿화
3. **단계적 도입**: 1~2개 프롬프트부터 단계적 도입
4. **팀 공유**: 팀 내 공유

### 학습 효과 극대화
1. **대화 중시**: 대화형 사용을 유지
2. **기록 유지**: 요구 및 설계 기록 보존
3. **회복**: 프로젝트 종료 후 회고
4. **커스터마이즈**: 프로젝트 맞춤 커스터마이즈

## 🚨 자주 묻는 질문 (FAQ)

### Q1: 프롬프트가 너무 길어요.
**A**: 필요한 부분만 발췌하여 사용하십시오. 각 프롬프트는 별도의 섹션으로 구성되어 부분 사용이 가능합니다.

### Q2: 기술 스택이 다르면?
**A**: 프롬프트를 실행할 때 "내 기술 스택은 OO입니다. 그에 맞게 최적화해 주세요"라고 덧붙여 주세요.

### Q3: 개인 개발에서 Phase 1–2는 꼭 안 해도 되나요?
**A**: 소규모 개인 프로젝트에서는 선택 사항입니다. 그러나 학습 목적이나 복잡한 프로젝트에서는 실행을 권장합니다.

### Q4: 에러 발생 시?
**A**: `01-debug-cognitive.md`를 사용하십시오. 단순한 수정이 아니라 학습 중시의 디버그 프로세스로 근본적인 해결을 도모할 수 있습니다.

### Q5: 팀 도입 시 주의점은?
**A**: `examples/team_setup.md`를 참조하십시오. 단계적 도입과 역할 분담이 성공의 열쇠입니다.
