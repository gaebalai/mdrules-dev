주요 학습 내용
  • AI 에이전트란?: "행동하는 AI"의 정의와 기존 생성형 AI와의 차이점
  • RPA vs Agent: 규칙 기반 자동화(Level 1)와 자율적 추론(Level 3+)의 비교
  • 왜 지금인가?: LLM의 추론 능력 향상과 Tool Use의 결합
  • 유형 분류: 단일 에이전트 vs 멀티 에이전트 시스템
키워드 & 토론
자율성(Autonomy) 추론(Reasoning) RAG vs Agent 업무 자동화의 미래
💡 Insight: 단순 대화형 챗봇에서 업무를 완결하는 에이전트로의 진화를 이해합니다.
주요 학습 내용
  • 작동 원리: 인지(Perception) → 계획(Planning) → 행동(Action) 루프
  • 시스템 프롬프트: 에이전트의 페르소나와 제약 조건 설정 (ReAct 기법 등)
  • 메모리 시스템: 단기 기억(Context Window)과 장기 기억(Vector DB)
  • 도구 사용 (Tool Use): API 연동과 Function Calling의 이해
실무 적용 포인트

엔지니어를 위한 아키텍처 설계의 핵심 요소 분석

User: "지난달 매출 보고서 이메일로 보내줘"
Agent: [DB 조회] -> [데이터 요약] -> [이메일 작성] -> [전송 API 호출]
주요 사례 분석
  • SW 개발: Devin, GitHub Copilot Workspace (코딩부터 배포까지)
  • 엔터프라이즈: Salesforce Agentforce (고객 대응 자동화)
  • PC 조작: Claude Computer Use (UI 조작 자동화)
  • 개인 비서: 퍼스널 에이전트의 진화
AgentOps (운영)
  • 모니터링: 에이전트의 행동 추적 및 로그 분석
  • 평가(Eval): 할루시네이션 방지와 성능 측정 지표
  • 비용 관리: 토큰 사용량 최적화 전략
Innovation Ops 전략
  • 도입 로드맵: 스몰 스타트에서 전사 확산까지
  • 인간 협업: Human-in-the-loop (인간 개입) 설계의 중요성
  • 보안 및 권한: 자율적 에이전트의 접근 권한 통제
  • 조직의 변화: 지시하는 관리자에서 설계하는 관리자로
미래 전망 (Future)

AGI(범용인공지능)로 가는 길목에서의 에이전트 경제(Agent Economy) 출현과 기업의 준비 사항 논의.
